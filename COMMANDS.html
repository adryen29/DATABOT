Discord DB Bot

Command reference — structure & data language

How commands work

Every command is sent as a normal Discord message starting with your bot's prefix (default !db), for example:

!db CREATE CATEGORY Logs

Commands come in two kinds:

  • structure — actually creates/renames/deletes real Discord categories & channels.
  • data — reads/writes fields stored per-channel, in a local encrypted database (not in Discord messages).

Only server members with Administrator permission can run commands.

Structure commands

CREATE CATEGORY name structure

CREATE CATEGORY Logs

Creates a new Discord category named Logs.

CREATE CHANNEL name IN category structure

CREATE CHANNEL FirstTicket IN Logs

Creates a text channel called FirstTicket inside the Logs category.

RENAME CATEGORY name TO newName structure

RENAME CATEGORY Logs TO Archive

RENAME CHANNEL name TO newName structure

RENAME CHANNEL FirstTicket TO SecondTicket

DELETE CATEGORY name structure

DELETE CATEGORY Logs

Deletes the category from Discord and removes its record (and all its channels' data) from storage.

DELETE CHANNEL name structure

DELETE CHANNEL SecondTicket

LIST CATEGORIES structure

LIST CATEGORIES

Lists every category the bot knows about.

LIST CHANNELS IN category structure

LIST CHANNELS IN Logs

Data commands

SET channel field = value data

SET FirstTicket subject = Cant log into my account SET FirstTicket email = adryen@example.com SET FirstTicket password = hunter2

Stores a field on a channel's record. The field's name decides how it's protected — see the Security table below. Multi-word values don't need quotes; everything after = is taken as the value.

GET channel [field] data

GET FirstTicket subject GET FirstTicket

With a field name, returns just that field. Without one, returns every field set on that channel. Password-type fields never return their real value here — they always show as hashed/masked.

UNSET channel field data

UNSET FirstTicket subject

Removes a field entirely. Alias: DELETE FIELD FirstTicket subject.

VERIFY channel field value security

VERIFY FirstTicket password hunter2

The only way to check a password-type field — compares the given value against the stored hash and replies with a match/no-match result. It never reveals the real stored value.

Field security reference

How a field is stored depends on its name, configured in src/schema.js:

Field namesStorage methodRetrievable via GET?
password, pass, pwdbcrypt hash (one-way)No — use VERIFY
email, bio, address, phone, note, notesAES-256 encryptedYes — decrypted automatically
subject, status, username, role, and any other nameStored as plain textYes

Add any field name to src/schema.js to reclassify it — for example, adding "ssn": "encrypt" would encrypt any field literally named ssn.

Good to know

Data lives in data/db.json, encrypted/hashed at rest — not in Discord message history. Each channel also shows a pinned summary embed for convenience, but that embed is just a display; all real reads and writes go through the database file. Back up your ENCRYPTION_KEY somewhere safe — losing it makes every encrypted field permanently unreadable.

Example: leaderboard use case

CREATE CATEGORY Leaderboard CREATE CHANNEL PlayerOne IN Leaderboard SET PlayerOne score = 4200 CREATE CHANNEL PlayerTwo IN Leaderboard SET PlayerTwo score = 3900 LIST CHANNELS IN Leaderboard GET PlayerOne score

For automatic sorting/ranking (e.g. a TOP Leaderboard score 10 command), ask to have it added — it's a small addition to the parser and dispatcher.
